# Архитектурный план Pinory v1

## Границы

`apps/miniapp` отвечает за Telegram WebView и браузерный режим. `apps/api` — единственная доверенная граница для авторизации, видимости и изменений. `apps/bot` открывает Mini App, обрабатывает deep links и доставляет уведомления. `packages/shared` хранит DTO, enum и Zod-схемы, `packages/config` — бренд и продуктовые настройки.

## Поток основного сценария

1. Mini App передаёт неизменённый `initData` в `/auth/telegram`.
2. API проверяет HMAC и `auth_date`, upsert-ит пользователя и выдаёт короткий JWT.
3. Карта запрашивает только текущий bounding box. SQL применяет PostGIS-индекс и серверные правила видимости.
4. При создании отметки API в транзакции создаёт/использует Place и MapEntry, проверяет idempotency key.
5. Фотографии проверяются, очищаются и преобразуются в три WebP-варианта, затем сохраняются в S3.
6. Повторный запрос карты получает сохранённую запись из PostgreSQL.

## Расширение до v2

Таблицы location sessions/samples/visits создаются отдельной миграцией и не используются в v1. Источники координат реализуют интерфейс `LocationSource`; Mini App предоставляет только foreground-координату.
